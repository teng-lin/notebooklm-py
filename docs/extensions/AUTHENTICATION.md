# NotebookLM 认证指南

## 🔐 认证方式

NotebookLM 使用 **浏览器内授权**，无需 API 密钥。

## 📋 认证流程

### **首次使用**

1. **运行脚本**
   ```bash
   python your_script.py
   ```

2. **自动打开浏览器**
   - 脚本会自动启动 Playwright 浏览器
   - 导航到 NotebookLM 登录页面

3. **登录 Google 账号**
   - 输入您的 Google 账号和密码
   - 完成两步验证（如果启用）

4. **授权访问**
   - 允许脚本访问 NotebookLM
   - 浏览器会保存会话信息

5. **开始处理**
   - 认证完成后，脚本自动开始处理视频

### **每日使用**

⚠️ **重要提示**：浏览器会话每天会过期

**每天首次运行时**：
1. 脚本会检测会话是否过期
2. 如果过期，会自动打开浏览器
3. 重新登录 Google 账号
4. 继续处理

**同一天内多次运行**：
- ✅ 会话仍然有效
- ✅ 无需重新登录
- ✅ 直接开始处理

## 🔧 故障排除

### 问题 1：浏览器无法打开

**可能原因**：Playwright 未安装

**解决方法**：
```bash
# 安装 Playwright
pip install playwright

# 安装浏览器驱动
playwright install chromium
```

### 问题 2：登录后立即过期

**可能原因**：浏览器数据未保存

**解决方法**：
1. 检查浏览器数据目录权限
2. 确保脚本有写入权限
3. 尝试以管理员身份运行

### 问题 3：无法访问 NotebookLM

**可能原因**：
- 网络问题
- Google 账号权限问题
- NotebookLM 服务不可用

**解决方法**：
1. 检查网络连接
2. 确认 Google 账号可以访问 NotebookLM
3. 在浏览器中手动访问 https://notebooklm.google.com

## 💡 最佳实践

1. **使用专用 Google 账号**
   - 建议使用专门的账号用于自动化
   - 避免影响个人账号的使用

2. **定期检查会话**
   - 每天首次运行时留意登录提示
   - 确保认证成功后再离开

3. **保持浏览器数据**
   - 不要删除浏览器缓存目录
   - 可以减少重新登录的频率

## 🔒 安全注意事项

1. **不要共享浏览器数据**
   - 浏览器数据包含您的登录信息
   - 不要将其提交到 Git 或分享给他人

2. **使用强密码**
   - 确保 Google 账号使用强密码
   - 启用两步验证

3. **定期检查授权**
   - 在 Google 账号设置中检查授权的应用
   - 移除不再使用的授权
